## Quick Deploy

### Step 1 вЂ” Database (Neon)
1. Create project at neon.tech
2. Open SQL Editor в†’ paste + run `schema.sql`
3. Copy your connection string

### Step 2 вЂ” Render Blueprint
1. Push to GitHub
2. Render в†’ New в†’ Blueprint в†’ connect repo
3. Both services (`ngo-lms-api` + `ngo-lms-client`) are created automatically

### Step 3 вЂ” Environment Variables

**Backend (`ngo-lms-api`):**
| Key | Value |
|---|---|
| `NODE_ENV` | `production` |
| `DATABASE_URL` | Your Neon connection string |
| `JWT_SECRET` | Auto-generated by Render |
| `FRONTEND_URL` | Your frontend URL (add after first deploy) |

**Frontend (`ngo-lms-client`):**
| Key | Value |
|---|---|
| `VITE_API_URL` | Auto-set from backend via `fromService` |

### Step 4 вЂ” Verify
- `https://your-api.onrender.com/` в†’ `{"status":"ok"}`
- `https://your-api.onrender.com/health` в†’ `{"status":"healthy","db":"connected"}`
- Frontend loads landing page в†’ login works

## Common Errors
| Error | Fix |
|---|---|
| `Cannot GET /` | Fixed вЂ” `/` and `/health` routes exist |
| `ERR_MODULE_NOT_FOUND` | `rootDir: backend` in render.yaml, `npm install` as buildCommand |
| CORS error | Set `FRONTEND_URL` on backend and redeploy |
| Charts not rendering | `chart.js` + `react-chartjs-2` in dependencies вЂ” runs after `npm install` |
| Blank page on refresh | `_redirects` + `routes` in render.yaml handle SPA routing |
| 401 after redeploy | Don't regenerate `JWT_SECRET` вЂ” keep a stable value |
